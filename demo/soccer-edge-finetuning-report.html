Executive conclusion
We have built a reproducible, rights-safe training and evaluation pipeline. The highlight-only CV aggregates do not yet provide reliable match-outcome prediction. The open-event experiment is materially stronger, which points directly to the next phase: representative full-match footage and time-aligned event/tracking supervision.

1. What the platform does

Soccer-Edge ingests open event/tracking data and approved local video, normalizes them into lineage-tagged Parquet tables, generates computer-vision and event features, trains tabular and CNN models, evaluates them on held-out matches, and blocks promotion when evidence is insufficient.

approved sources → normalized tables → detections / state → features → models → calibration → promotion gate

Local footage citation rule
The 98 highlight files are permitted local source assets, not public redistributable URLs. The repository records rights_status=licensed and a rights reference in the data card. Public URLs remain discovery metadata only and are never processing inputs.

3. What was trained or fine-tuned

Object-detection path

YOLO-based frame detections feed frame manifests, low-confidence queues, crop review, corrected annotations, grouped train/validation splits, YOLO dataset layouts, optional object-model training, object metrics, and confusion matrices.

Tabular match model

Match-level CV features: player count, ball count, detection rate, and normalized ball-center coordinates. The path trains a calibrated winner classifier plus home/away score regressors.

CNN occupancy-grid model

Frame detections are rasterized into player/ball occupancy grids. Grouped sliding windows produce NPZ tensors for a CNN winner model evaluated at sequence and match levels.

Realtime research path

Rolling state, lightweight nearest-centroid tracking, low-confidence review, possession/territory/pressure proxies, smoothed expected-winner probability, and momentum/comeback triggers.

4. The 98-match case study

The processed study contains 98 matches and 50 teams. It generated 88,127 detections: 86,007 player detections and 1,541 ball detections, plus 7,878 occupancy-grid frame rows. Match labels were parsed from supplied local filenames into a structured results table.

Concrete example: HL001

Argentina 3–1 Switzerland · 150 analyzed frames · 1,567 raw detections · 1,514 person detections · 45 sports-ball detections. The existing HL001 downstream track/player/ball-state tables are empty, so the team demo correctly presents this as frame-level detection evidence rather than completed possession or tactical tracking.

The repeated-CV CNN result was approximately 49.3% ± 1.8% sequence accuracy, 48.9% ± 1.3% match accuracy, and 0.630 ± 0.011 Brier. These results do not justify promotion.

Why the open-event model matters

The project paper records a separate StatsBomb Premier League 2023/24 event-data evaluation over 380 matches: 63.3% ± 2.9% accuracy, 0.164 Brier, score MSE 0.96 / 0.87, and a 41.3% majority baseline. This is the clearest evidence that the evaluation stack can find useful signal when supplied with rich event supervision.

6. What the work proves — and does not

Proven now

Rights gating by construction; heterogeneous data ingestion; lineage preservation; frame/detection/annotation artifacts; CV and event features; tabular and CNN training paths; held-out evaluation; calibration metadata; promotion gating.

Not proven yet

Reliable player identity, stable tracking, arbitrary-view pitch calibration, predictive lift from highlights, production-grade live prediction, profitability, or suitability for real-money execution.

7. Recommended next phase

  1. Secure written rights for representative full-match local footage.
  2. Record period and match-second alignment in the manifest.
  3. Manually validate detector outputs on a frame sample.
  4. Correct player/ball labels and fine-tune the local object model.
  5. Add stable tracking and pitch calibration.
  6. Join CV state to event timestamps.
  7. Train leakage-safe in-play features and future-window labels.
  8. Compare against event-only, Elo, and market baselines.
  9. Promote only if calibration and out-of-sample lift pass the gate.
